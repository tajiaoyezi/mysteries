### C5 · 工具卡(`kind: tool`)
- 字段:`{name, args, readonly, status: running|done|error, output[], exit, truncated, extra}`。
- 驱动:`ToolCallStarted`(running)→ `ToolCallFinished`(done / error)。
- **头**:状态 glyph(`running`→spinner / `done`→`✓` / `error`→`✗`)+ `name`(bold,accent)+ `args`(dim);只读卡右侧徽章 `只读 · 自动运行`。
- **体**:`output` 行(等宽 `text.body`);截断时 `⋯ +N 行已截断 (max_output_bytes)`(对应 §5.3 `ToolOutcome.truncated`)。
- **脚**:`exit {code}`(dim;非 0 用 `error.fg`)。
- `error` 态 output 示例:`权限被拒绝 (user denied)`。
- 渲染:✅ 全结构;spinner ⚠️(braille / 点阵帧动画)。

### C6 · 权限框(`pending: {tool, args, diff}` · status `WaitingForPermission`)
- 驱动:`PermissionRequired`(§3 携 `oneshot`)→ `y`/`n` 经 oneshot 回送。**自动放行命中时(`auto_allows`)不产生 `pending`,本组件不渲染。**
- 位置:钉在状态行上方(与 §8 一致)。`warning` 色框。
- **头**:`▲ 需要授权  {tool}  {args}`。
- **体**(写 / 编辑):**diff**,行类型 `ctx`(dim,前导空格)/ `add`(`success`,`+` gutter)/ `del`(`error`,`-` gutter——原型未演示但实现需有);`run_shell` 类显示命令、无 diff。
- **动作行**:`[y · 允许]   [n · 拒绝]`(按钮→文本 ⚠️)+ 提示 `Enter = 允许 · Esc = 拒绝`。
- **拒绝路径**:tool 卡转 `error` + notice「已拒绝 — 一条 is_error 的 ToolResult 已注入 history,模型据此改路线,而非静默跳过」(§5.4)。
